Where should PM artifacts live on the site?

You have user stories, the AMVero presentation, an onboarding doc, and more. Where does a recruiter encounter this depth?

A

Expand the existing work cards

Each product card gets a second layer — click "See PM work" to reveal user stories, the onboarding doc, and other artifacts directly below the current metrics + quote block. No new section, depth lives inside the product.

B

New dedicated "PM Craft" section

A standalone section after Selected Work, focused entirely on how Michael works as a PM — user stories, requirements docs, onboarding artifacts, structured as a showcase of craft rather than attached to a specific product.

C

Surface highlights inline, link to full docs

Don't add a new section or new layers. Instead, pull the most compelling excerpt from each artifact — one user story, one onboarding principle, one slide — and weave them directly into the existing cards as proof points. Full docs linked for those who want more.

D

Add to the existing modals

The AMVero modal already has the interactive prototype. Add a "PM Artifacts" tab inside the modal — user stories, the onboarding doc, presentation highlights — so the depth is one more click from where recruiters already go.
