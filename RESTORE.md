## Overview

This is the DEVELOPMENT / EXPERIMENT repository for Michael Korenevsky's PM portfolio site. It is a Next.js 16 App Router site with a dark theme, bento-grid layout, and skill-driven development via Claude Code.

## Prerequisites

- Node.js 22+ and npm/pnpm
- Vercel CLI (`npm i -g vercel`)
- Claude Code CLI
- Access to Michael's Google Drive portfolio folder
- Figma account with Dev Mode enabled (for design reference)
- Git with SSH key for `github-personal` host alias

## Step 1: Clone and install

```bash
git clone git@github-personal:korm85/ClaudeCode-portfolio-dev
cd ClaudeCode-portfolio-dev
npm install
npm run dev    # → http://localhost:3000
```

## Step 2: Configure Claude Code environment

```bash
# Global CLAUDE.md
cp .claude-backup/global-CLAUDE.md ~/.claude/CLAUDE.md

# settings.json (edit NVM paths to match your Node install)
cp .claude-backup/settings.json ~/.claude/settings.json

# mcp.json (fill in FIGMA_API_KEY before copying)
cp .claude-backup/mcp.json ~/.claude/mcp.json

# Hooks
cp -r .claude-backup/hooks/. ~/.claude/hooks/
chmod +x ~/.claude/hooks/*.sh

# Memory
mkdir -p ~/.claude/projects/-home-<user>/memory/
cp -r .claude-backup/memory/. ~/.claude/projects/-home-<user>/memory/
```

## Step 3: Set up MCP servers

### Google Drive MCP
```bash
claude mcp add google-drive npx @modelcontextprotocol/server-gdrive
```
Authenticate with the Google account that has access to the portfolio Drive folder:
`https://drive.google.com/drive/folders/184rMRm2T_1RM4CjZvGG_wtHHoHYS4Ip2`

### Figma MCP
```bash
claude mcp add figma-dev-mode --transport http http://127.0.0.1:3845/mcp
```
Requires: Figma desktop app running with Dev Mode MCP enabled in Figma settings.
Get a new FIGMA_API_KEY from figma.com → Account Settings → Personal Access Tokens.

## Known Gotchas

- **`BRIEF.md` overrides Bento design skill** — if Bento and BRIEF.md disagree on a visual detail, BRIEF.md wins
- **Always invoke `portfolio-design-system` skill before any UI change** — required by AGENTS.md
- **Content source of truth is `docs/product-case-studies.md`** — never write a metric not in that file
- **Never blur AI vs physics** — AMVero is AI (computer vision), Simulation is physics-based, never say "AI Simulation"
- **This repo deploys to the dev Vercel project, not `themishka.me`** — the official deploy happens via GitHub push to `michael-korenevsky-portfolio` repo
- **Node.js path in hooks** — `settings.json` has hardcoded nvm paths. Update to match your local Node install path.
